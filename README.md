CivisID is a robust Java-based Command Line Interface (CLI) application designed to simulate the backend logic and data management principles of a government-level citizen registration system. It serves as a foundational prototype for e-governance tools, focusing on accuracy, data integrity, and retrieval speed.

While many introductory console applications are ephemeral‚Äîlosing all user input the moment the program terminates‚Äîthis project implements persistent data storage using the CSV (Comma Separated Values) format. This critical design choice ensures that data is not only saved permanently but is also instantly portable and interoperable. By adhering to standard CSV protocols, the government records generated by CivisID can be seamlessly opened, analyzed, and visualized in external spreadsheet software like Microsoft Excel, Google Sheets, or LibreOffice Calc without requiring complex database drivers.

üöÄ Key Features and Logic

This system moves beyond simple data entry forms by implementing sophisticated conditional logic and strict validation rules to ensure high data integrity standards.

Context-Aware Relative Logic:
The system replaces static, one-size-fits-all forms with dynamic, context-sensitive prompts. By intelligently analyzing the user's Gender and Marital Status inputs in real-time, the application adapts to request the specific relative name required by administrative protocols. This mirrors real-world documentation standards:

Married Male: The interface prompts specifically for "Wife's Name", ensuring spousal records are linked correctly.

Married Female: The interface prompts specifically for "Husband's Name".

Unmarried Citizens: The system defaults to "Father's Name" logic, maintaining standard lineage tracking for single individuals.

7-Digit Unique ID Generation:
To simulate a secure registry, the system utilizes a randomization algorithm to generate a unique 7-digit National ID for every new applicant. Unlike sequential IDs (e.g., 001, 002), which can be predicted, this randomized approach enhances anonymity and security. It also allows for efficient indexing and hashing of records within the internal database.

Robust Data Persistence (CSV):
The application features a dedicated custom file handler that manages the entire storage lifecycle, ensuring data safety during program execution.

Smart Headers: The system checks the storage directory upon launch; if the file is new or missing, it automatically initializes it with the appropriate column headers (Name, Age, Relation, etc.), preserving data structure.

Append Mode Safety: New records are safely appended to citizens.csv using file locking mechanisms where applicable. This prevents the accidental overwriting or corruption of existing historical data that can occur with simpler file-writing methods.

Full CRUD Lifecycle:
The system supports the complete management lifecycle of citizen profiles, giving administrators full control:

Create: Register new citizens with rigorous input validation (e.g., preventing text input for Age or Family Members) to prevent database pollution.

Read: Instant retrieval of citizen details using their unique ID, allowing officials to verify registration status in milliseconds.

Update: Provides functionality to correct specific mutable fields‚Äîsuch as Name, Address, or Phone Number‚Äîwithin the active session, ensuring records remain current.

Delete: Grants administrative capability to remove invalid, duplicate, or deceased records from the active registry memory, maintaining database hygiene.

Comprehensive Data Capture:
Designed specifically for the needs of e-governance, the system captures 15+ distinct data points. This includes Contact Info (Email/Phone) for government notifications, Pincode for regional demographic sorting, Family Member counts for census and resource tracking, and ID Proof numbers for cross-verification against legacy systems.
